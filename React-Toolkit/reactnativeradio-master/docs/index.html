A) What You Should Know as Buyer - top

You must have React Native knowledge or you must me inclined to learn it.

You also need to be able to install React Native and its dependencies on your machine. To do that I recommend you to use the React Native official installation guide here.

And last, but not least, in order to modify the app template, you must have knowledge in mobile technologies such as JavaScript, Android Studio, XCode.

B) YARN Installation - top

To install your project dependencies, you must first install YARN on your system. From there you can go to the Installation docs.

YARN "Getting Started" page: here.

With YARN installed, you can proceed to the next step.

C) Install Project Dependencies - top

With YARN installed, you need to install the project dependencies. To do that you should open the command prompt (Windows), or terminal (Mac OS) ON YOUR PROJECT FOLDER and type the following commmand:

yarn install

D) Finish Project Dependencies (Mac OS) - top

If you are on a Mac OS, you need to finish installing your project dependencies by running the following commands on terminal (from your PROJECT FOLDER):

cd ios
pod install

E) Radio Player Function - top

This app template was built with a radio player from react-native-track-player plugin. Here are some details about it to get you started:

  1. Starting radio stations available are inside "{projectName}/src/states/radio.state.js" file
  2. To add more radio stations you should gather the station name, image and AACP url
  3. The AACP url can be obtained by installing an extension named Video DownloadHelper on you web browser
  4. You can customize background play settings by working on the audioBackgroundBehavior function inside "{projectName}/src/actions/radio.action.js" file
  5. For more details about the react-native-track-player plugin, click here
    1. IMPORTANT: To allow background audio playback on iOS, you need to activate the "Audio, Airplay and Picture in Picture" background mode in Xcode. Without activating it, the audio will only play when the app is in the foreground, as described here

F) Localization Support - top

This app template currently has enUS language, but you can simply add more languages at the "{projectName}/src/languages" folder if you want. To do that, add a "xxYY.language.js" file (similar to the "enUS.language.js") and export it on "{projectName}/src/languages/languages.js" file.
